# AIエージェント最小仕様（MVP）

## 目的と対象タスク例
初学者が安全に扱える「半自動エージェント」を想定する。対象タスクは軽量で再現可能なものに限定する。
- 要約（長文の要点抽出）
- チェックリスト作成（作業手順の整理）
- ログ原因候補の列挙（事実ベースで推測）
- フォルダ整理の提案（削除や移動は提案止まり）

## 入出力
- 入力: テキスト / ファイル / URL
- 出力: 固定フォーマット
  - Summary
  - Plan (Dry-run 時のみ)
  - Actions (Run 時のみ)
  - Evidence (参照したファイル/URL/ログ)
  - Risks (注意点)

## コア機能
- Planner: 手順を提案し、Dry-run を構成する
- Executor: ユーザーが明示した手順のみ実行する
- Reporter: 結果を固定フォーマットで報告する
- Guard: 禁止事項と権限を監視する
- Store: ログと重要な決定事項を保存する

## ツール（最小）
- ファイル read / list
- Web GET
- 簡易変換（フォーマット整形など）
- 全文検索

## 安全柵
- 実行モードは Dry-run / Run の2段階
- 権限は read/list/get を中心にし、書き込み/削除/外部投稿はデフォルト禁止
- 不確実な操作は提案に留める

## 期待性能
- 1タスクあたり 3〜7 ステップを目安
- 失敗時は「理由・代替案・必要な追加情報」を提示

## 評価
固定10ケースで評価し、以下の観点を確認する。
1. Dry-run の提示がある
2. Run がユーザー指示に限定される
3. 出力フォーマットの順守
4. 根拠が明記される
5. 禁止事項を回避
6. 目的とズレがない
7. 変更が最小
8. 不確実性の明示
9. ログが残る
10. 再現性がある

## 推奨アーキテクチャ
Router / Planner / Executor / Reporter / Guard / Store

## 初学者向け完成形の例（Tech調査エージェント）
- 入力: 調査テーマ、参考URL
- Dry-run: 調査手順と確認観点を提示
- Run: 指示されたURLのみ参照し要約
- 出力: Summary / Evidence / Risks を必ず含める
